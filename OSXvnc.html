OSXvnc 1.11

Frequently Asked Questions

(Last Modified: 18 January 2003)

 

 

Q: How can I start up OSXvnc from a command line?

A: To startup OSXvnc from command line go inside the Application directory(OSXvnc.app) and launch the OSXvnc-server process.  To change parameters you will need to give it arguments (-rfbport to set port, -vncauth to specify a password file, etc).  For usage run the command with -help.

 

Q: I'm running a headless OS X machine and OSXvnc doesn't work.

A: We are aware of a problem when some OS X machines boot up they will disable certain video functions when no monitor is plugged in.  We are looking at ways to solve this in OSXvnc but for the time being you can purchase a dongle that makes the OSX machine think that a monitor is plugged in.

 

Q: How can I setup OSXvnc to startup as a service without needing a user logged in?

A: OS X uses folders inside the /Library/StartupItems folder to launch services at runtime.  An OSXvnc startup folder is included inside the OSXvnc.app/Contents/Resources directory or one can be downloaded here.  Modify the OSXvnc script (inside the directory) to suit your needs and then copy the folder into /Library/StartupItems, you will need admin permissions.  We hope to have better integration for this process in an upcoming version of OSXvnc, ideally through a System Preference pane.

 

Q: I've setup OSXvnc as a service but when a user logs out it breaks the server.

A: This should be fixed with the 1,11 version of the the startup script.  This is a temporary workaround, we are aware of the real problem, and know where it is in the code, but don't yet have a fix for it.  As soon as we have a better fix we'll issue a corrected server.

 

Q: What happened to the storepasswd and OSXvnc-server executables?

A: The OSXvnc-server and storepasswd executables are now stored back in the /Contents/MacOS directory and are linked from the root OSXvnc.app directory as well as the /Contents/Resources directory, this prevents the GUI from launching a second dock icon when they are executed.

 

Q: I'm using Chicken of the VNC and my screen doesn't refresh

A: This should be fixed in 1.11.  We are aware of an incompatibility between OSXvnc1.0 and some versions of Chicken.

 

Q: I am using an International keyboard (or keyboard layout) and the characters don't come up appropriately?

A: Currently OSXvnc doesn't handle this as well as we would like, part of the problem is that if both the remote and local users are using an international keyboard layout the keystrokes actually get double-corrected thus producing the standard US keys, try setting one of the keyboards back to US, this works a bit better.

 

Q: Is there a Web Server with OSXvnc or a Java applet integrated into OSXvnc.

A: No, but you can download a java applet client and put it somewhere on the OSX machines web served directory.  Then enable web serving on your machine and access the java applet.  We didn't port the java applet but a copy of it can be downloaded here.  

 

Q: How can I have my connection to OSXvnc tunnel through SSH?

A: Since OS X ships with SSH support built in this is pretty easy.  First you will want to make sure your VNC machine has SSH running, you can turn this on in System Preferences -> Sharing Panel and check the box for "remote login".  Then you need an SSH client on the machine you want to connect from (again it's there by default on OS X). For the rest of the instructions read the VNC page http://www.uk.research.att.com/vnc/sshvnc.html

 

Q: What are you looking at adding for future versions?

A: Probably highest on our list are making the mouse cursor (optionally) appear, startup/service behavior, and including the Java applet so you can connect through a browser.

 

Q: OSXvnc is open source so....Where's the source?

We have had numerous problems getting our repository transferred over to SourceForge, and thought it was fixed,  We finally have the full source transferred over, please try it now at: http://sourceforge.net/projects/osxvnc
